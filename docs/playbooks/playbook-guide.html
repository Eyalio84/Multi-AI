What Are Playbooks?

Playbooks transform documentation into executable recipes. They show:

  • WHAT combinations work
  • WHEN to use them
  • HOW to implement
  • WHY (cost/speed/quality trade-offs)

The Philosophy

Playbooks follow the NLKE v3.0 methodology: Build WITH AI, Document AS you build. Each playbook is:

Self-Contained

Everything needed to implement a capability in one document.

Cross-Referenced

Links to related playbooks for deeper exploration.

Standardized

Consistent structure for predictable navigation.

Playbook Structure

Every playbook follows this standard anatomy:

1. Header Metadata 
# Playbook N: Title
## Subtitle describing the capability

**Version:** 1.0
**Last Updated:** November 2025
**Target Audience:** [Who should read this]
**Difficulty:** [Beginner/Intermediate/Advanced]
**Time to Implement:** [Estimate]
**Value:** [What you'll achieve]
**Related Playbooks:** [Links to related PBs]
2. Table of Contents 
## Table of Contents

1. [Overview](#overview)
2. [Strategic Context](#strategic-context)
3. [Core Patterns](#core-patterns)
4. [Implementation Guide](#implementation-guide)
5. [Decision Framework](#decision-framework)
...etc
3. Overview Section

Always use ## Overview (not "Introduction") for consistency.

Include:

  • What You'll Learn
  • Prerequisites
  • Key concepts introduction
4. See Also Section 
## Additional Resources

**See Also (Playbooks):**
- **[Playbook N: Name](PLAYBOOK-N-NAME.md)** - Brief description
- **[Playbook M: Name](PLAYBOOK-M-NAME.md)** - Brief description
5. Footer Metadata

Creating a New Playbook

Step 1: Identify the Gap

Before creating a new playbook, ask:

  • Does this capability exist in another playbook?
  • Could this be a section in an existing playbook?
  • Is this substantial enough for a full playbook (~500+ lines)?
  • Does it follow the "executable recipe" philosophy?

Step 2: Choose Your Number

NumberCategoryNotes
0MetaReserved for playbook system
1-3FoundationsCore API patterns
4-6AdvancedDomain-specific
7MetaFractal discovery
8-11IntegrationSystem connectivity
12+ExtensionsFuture growth

Conventions & Standards

Naming

PLAYBOOK-N-TITLE-WITH-DASHES.md

All caps, dashes between words.

Section Names

Use "Overview" not "Introduction"

Use "See Also (Playbooks):" for links

Links

Relative paths: PLAYBOOK-N-NAME.md

Full format: **[PB N: Name](file.md)**

Headers

Always include Version, Date, Audience, Related Playbooks

Quality Checklist

  • Has all required header metadata
  • Has Table of Contents with working links
  • Uses "Overview" section (not Introduction)
  • Has "See Also (Playbooks):" section
  • Has footer metadata
  • At least 3 related playbook links
  • Includes real-world examples
  • Has troubleshooting section
  • Added to PLAYBOOK-INDEX.md
