**[Full Playbook](PLAYBOOK-1-COST-OPTIMIZATION.md)**

**Best for:** High-volume API usage, document Q&A, evaluation workflows, batch processing
**Key patterns:** Cache breakpoints, batch request formatting, compound optimization
**Prerequisites:** Claude API access, basic API knowledge
**Common pitfalls:** Cache invalidation, batch timeout handling, incorrect cache sizing
**Integration:** Foundational for PB 3 (agents), PB 4 (KG), PB 6 (ML)

## Playbook 24: 8-Step Implementation Workflow ⭐ META-PLAYBOOK

**The failsafe pattern that made Phase 1 successful**

**Use When**: Any complex implementation task (3+ files, 500+ LOC, multiple components)
**Key Value**: Gate-based progression prevents failure, captures learnings, feeds forward to next phase
**Status**: ✅ Validated by Phase 1 (100% success rate)

### The 8 Steps

1. **Phase Kickoff (AskUserQuestion)** - Clarify requirements before coding
2. **Implementation** - Write code efficiently without distractions
3. **Testing** - Validate functionality systematically (unit → integration → user)
4. **Debugging** - Fix issues discovered during testing
5. **Edge Cases** - Prevent production failures (empty inputs, corruption, concurrency)
6. **Documentation Upgrade** - Ensure knowledge transfer (README, architecture docs)
7. **--deep Save (Memory System)** - Capture learnings for compound growth
8. **Pattern Extraction & Mermaid Update** - Feed learnings back into methodology

### Failsafe Mechanisms

- **Front-loads validation** (Step 1 prevents building wrong thing)
- **Separates concerns** (code → test → debug isolated)
- **Captures learnings** (Step 7 prevents knowledge loss)
- **Feeds forward** (Step 8 improves future iterations)
- **Gate-based** (can't skip steps)

### Integration

Combines with:
- **P8 (Continuous Learning)**: Step 7 feeds fact persistence

- **P11 (Session Handoff)**: Step 7 enables seamless handoffs
